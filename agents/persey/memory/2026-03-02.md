## CodeMirror 6 bleed fix (React version)

Root cause: `.pane-cm` was `flex:1` but not `display:flex;flex-direction:column` — so CM's internal `height:100%` couldn't resolve against a flex-sized parent → scroller grew unbounded → text painted outside pane into sidebar.

Fix applied to `client/src/index.css`:
- `.pane-cm` → add `display:flex;flex-direction:column;min-height:0`
- `.cm-editor` → `flex:1;min-height:0;overflow:hidden` (not `height:100%`)
- `.cm-scroller` → add `contain:strict` (hard paint containment)
- `.cm-content` → add `overflow:hidden`

This is the CORRECT fix for CodeMirror 6 in a flex layout. Remember for future CM6 integrations.

## Editor pane fixes (commit 579fcf7)

### Bug: File content not rendering after click
**Cause:** `cmRef` div had `style={{ height: '100%' }}` inside `.pane-cm` which is `display:flex;flex-direction:column`. `height:100%` in a flex container can resolve to 0. Also, `.cm-editor{flex:1}` requires its parent to be a flex container — the `cmRef` div wasn't one.

**Fix:** Changed cmRef div to `style={{ flex:1, minHeight:0, display:'flex', flexDirection:'column', overflow:'hidden' }}` so CodeMirror properly fills the pane.

### Bug: Files show as dirty/modified immediately after loading
**Cause:** CodeMirror's `updateListener` fires on ANY doc change including programmatic dispatches (like loading file content). This called `updateContent()` → set `isDirty:true`.

**Fix:** Added `externalUpdate = useRef(false)` flag. Set to `true` before dispatching external content, back to `false` after. `updateListener` skips `updateContent` when flag is set.

### Pattern to remember for CM6
- CM6 parent div needs `display:flex;flex-direction:column` for `.cm-editor{flex:1}` to work
- Use `externalUpdate` ref flag to distinguish programmatic vs user edits
- `contain:strict` on `.cm-scroller` prevents text overflow
