## RunMusic — Audio Crossfade Bug Fix (1:09 AM session)

### Root cause of "title changes but audio doesn't"
`crossfadeTo` in `useSpotifyPlayback.ts` used **stale `isPlaying` closure** — captured `false` at mount via `useCallback` deps, never updated. It fetched the new iTunes preview correctly but then checked `if (audio && isPlaying)` which was always `false` → skipped `audio.play()` entirely.

### Fix committed → f8824d1
- Added `isPlayingRef` (mirrors `isPlaying` state in real-time) to avoid stale closures
- `crossfadeTo` now checks `audioUnlockedRef.current` (real-time ref) instead of `isPlaying` state
- After fetching new preview, calls `audio.play()` whenever user has already unlocked audio
- `unlockAndPlay` now fires on **first tap anywhere** in story view (card 0 or later), not just swipe-to-card-1

### Key lesson
- Never use React state inside `useCallback` for real-time decisions — use a `useRef` mirror
- Pattern: `const isPlayingRef = useRef(false)` + `const setIsPlaying = useCallback((v) => { isPlayingRef.current = v; setIsPlayingState(v); }, [])`
- `audioUnlockedRef` is the "user has gestured" flag — if true, calling `audio.play()` won't be blocked by browser autoplay policy
