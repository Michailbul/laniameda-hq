## 2. X/Twitter — What's Working in 2026

Building on last week's deep research (see 2026-03-02.md). New findings from XLab complete guide:

### 6 Hook Frameworks That Consistently Perform
1. **Precise number:** "I sent 247 cold DMs to founders. Here's the only template that got a 68% reply rate." Specificity = credibility.
2. **Counter-intuition:** "Most creators post too often." Opens a loop instantly.
3. **"Did you know that..."** — 22M+ views across niches; creates curiosity gaps people feel compelled to close.
4. **"3 mistakes everyone makes about X"** — triggers self-identification; doubles engagement rates.
5. **Bold statement + proof demand:** Start with the claim, force the reader to seek the explanation.
6. **The result first:** Lead with the outcome, not the process.

### Profile as Step 0 (Critical — often skipped)
- **Bio formula:** "I help [target audience] achieve [specific result], [content hook or proof]"
- **Pinned tweet:** Best-performing thread or most representative value. Clear CTA.
- **Profile photo:** Real face > avatar. Neutral background, natural expression. Builds trust instantly.
- **Niche depth matters more than breadth:** "AI creative studio founder building in public" > "entrepreneur" or "tech person"

### The 70/30 Rule
- 70% of time: engage with others in your niche (replies, comments)
- 30% of time: post original content
Engagement with others is how the algorithm places you in topic clusters. Posting into the void doesn't work without community signal.

### What's Confirmed Dead in 2026
- External links in main tweet → near-zero distribution
- Essay threads (15+ tweets) → time decay kills before momentum
- Engagement bait phrases → Grok suppresses
- Generic filler → Grok semantically reads and downscores

## Claude Code — Advanced Workflows Research

### Agent Teams (experimental)
- Enable: `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS: "1"` in `~/.claude/settings.json`
- 7 primitives: TeamCreate, TaskCreate, TaskUpdate, TaskList, Task(team_name), SendMessage, TeamDelete
- Key difference from subagents: teammates can message each other directly (mailbox system)
- Teammates self-claim tasks via shared JSON at `~/.claude/tasks/{team-name}/`
- Delegate Mode (Shift+Tab): restricts lead to coordination only — critical for proper delegation
- Display: split-pane with tmux (`"teammateMode": "tmux"` in settings)
- Cost pattern: lead on Opus, teammates on Sonnet
- Best use: cross-layer features, parallel audits, competing hypotheses
- NOT worth it: single-repo features, sequential tasks, same-file edits

### Git Worktrees
- Flag: `claude --worktree <name>` — creates `.claude/worktrees/<name>/` + dedicated branch
- Each worktree = separate checkout, own branch, own git index — zero file conflicts possible
- Auto-cleanup: no changes → deleted on session end; changes → Claude prompts
- Gitignore: add `.claude/worktrees/` to each repo
- Key pattern: parallel experiments (two approaches to same feature simultaneously)
- Key pattern: feature + hotfix without stashing or losing context
- Key pattern: design system rollout across multiple repos simultaneously

### GSD Framework (Get Shit Done) — INSTALLED v1.22.0
- Installed globally to `~/.claude/` via `npx get-shit-done-cc --claude --global`
- Commands: `/gsd:new-project`, `/gsd:map-codebase`, `/gsd:discuss-phase N`, `/gsd:plan-phase N`
- Core mechanic: max 3 tasks per plan, each runs in fresh sub-agent (clean 200K context)
- Beats context rot: isolation per sub-agent, git commit per task, fully revertable
- Permissions workaround (root can't use --dangerously-skip-permissions): add Bash allowlist to `.claude/settings.json`
- Start new projects with `/gsd:new-project`, existing repos with `/gsd:map-codebase` first
- vs BMAD: GSD = solo/fast/no ceremony; BMAD = teams/enterprise/compliance (21 agents)
- Best next step: run `/gsd:map-codebase` on RunMusic
