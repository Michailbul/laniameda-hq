assistant: ## Tyler's Pipeline — Technical Breakdown

**How it works (step by step):**

1. **Input** — you type `/thread-to-carousel [topic]` in Claude Code. Accepts: raw topic, pasted tweet thread, or screenshots of tweets
2. **Voice/tone** — pulled from `CLAUDE.md` file in the project. Claude reads your brand voice before writing
3. **Thread writing** — Claude generates: hook tweet → body tweets → CTA. Fixed structure every time
4. **Slide breakdown logic** — Claude decides how to split tweets into slides:
   - Short tweet + short tweet → combined on one slide with divider
   - Long tweet (200+ chars) → its own slide
   - First slide always gets an image
5. **Image sourcing** (4 pipelines, in priority order):
   - User-provided images (highest priority)
   - Tavily web search (finds relevant images)
   - Live screenshot via Python script (good for tool screenshots)
   - Gemini image gen (for illustrations/diagrams)
6. **Config generation** — Claude builds a `config.json` with all slide data: profile info, theme, slide text, image paths
7. **Rendering** — Python script with Pillow renders each slide as 1080×1350 PNG:
   - Loads fonts, applies theme (light/dark)
   - Draws profile pic (circular mask), handle, verified badge
   - Wraps tweet text, centers vertically
   - Embeds images (rounded corners)
   - GIF slides → converted to MP4
8. **Output** — folder of PNGs + MP4s, ready to upload to Instagram

---

## Caveats in His Approach

- **Tavily dependency** — paid API key required for image search. Falls over without it
- **Gemini image gen** — separate API key, extra cost, not always needed
- **Python/Pillow = fragile** — pixel-perfect layouts break when text is longer/shorter than expected. Hard to iterate on design
- **No brand customization** — fixed layout, no way to swap themes or apply your own design language
- **Single platform format** — only does tweet/thread style. Can't produce LinkedIn carousels or other formats from the same pipeline
- **Manual image swap** — he literally goes to Google, downloads an image, pastes it in manually. Not automated
- **No memory** — every run starts fresh. Claude doesn't remember what worked before
- **No content repurposing hook** — just outputs slides, no connection to a publishing pipeline

| Manual image selection | Agent picks + ranks automatically, you only approve |
| CLAUDE.md voice config | laniameda brand rules baked into the skill |
| Standalone output | **Connects to content pipeline** — auto-tagged in gallery + queued for Notion |
